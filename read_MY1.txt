требования :
наличие python версии 3.6 и выше, tinedb
для запуска программ не требуется никаких дополнительных установок

 
 -base.json - тестовая база в формате tinedb
 -zap.txt - файл с тестовыми строками для проверки работоспособности функции распознавания запроса в формате :
 		текст запроса --- тип запроса --- ожидаемый ответ модуля распознавания
 -genlist.py - программа генерации базы данных base.json из 100 шаблонов с уникальными именами и уникальным набором пар поле-тип поляБ
 					а  также генерации запросов в файл  zap.txt
 -server.py - модуль разпознавания запроса. метод post должен получить запрос и в ответ послать символьную строку, содержащую либо 
 				имя шаблона  в случае нахождения такового, как описано в техническом задании, либо
 				строку в формате "{fildname1:type1,fildname2:type2,...fildnamen:typen}" 
 -test.py - программа проверки работоспособности модуля распознавания запроса.
 			при запуске построчно читает файл zap.txt,
 			выводит на экран :
 			-текст запроса,
 			-список катриджей (имя_поля,тип_поля) после анализа запроса
 			-ожидаемый ответ модуля
 			-тип запроса (direct - поля запроса и типы должны совпадать с шаблоном, имя которого ожидается,
 							add - полей в запросе больше, чем шаблоне, которому должен соответствовать запрос, дополнительные имена сгенерированы случайно
 							error - случайная генерация полей запроса, в большинстве случаев не должна иметь соответствующего шаблона)
 			-ответ модуля,
 			-результат сравнения - либо Совпадение, когда ответ идентичен ожидаемому, либо
 										НЕТ совпадения. Эта ситуация возникает в следующих случаях:
 														- тип add, имя шаблоа в ответе не совпадает с ожиданым именем: 
 																добавленные поля создали ситуацию, не описанную в ТЗ, 
 																когда запрос отвечает требованиям нескольких шаблонов
 														-Тип error, сгенерирован случайный запрос, для которого нашелся соответствующий шаблон
 			-выводится строка "Имя шаблона : " и программа переходит в состояние ожидания. 
 					пустой ввод - обработка следующего запроса,
 					ввод строки с именем шаблона порождает вывод запрошенного шаблона и переход в ожидание ввода


 					Прервать выполнение программы можно нажатием Ctrl-C
